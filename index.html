The Solution
Every agent gets a policy.
Every policy gets enforced.

Harbinger gives each agent a scoped token and enforces exactly what it can do. Agent requests go through a policy engine that validates identity, checks policy, and proxies with the real credential.

✓

Per-Agent Identity

Policy Engine
Per-agent policies.
Enforced at runtime.

Every agent gets its own policy. Define what it can access, which endpoints it can call, and when it's allowed to run. Policies are evaluated at the edge , before any request reaches your infrastructure.

✓

Agent → Group → Policy

Assign agents to groups. Policies apply to groups. One rule covers many agents.

✓

Time & Rate Limits

Deploy-bot can only run 9am-6pm. CI-runner: 500 requests/hour max.

✓

Path Restrictions

Agent X can only access /repos/myorg/deploy-*. Nothing else.

✓

AI-Powered Least Privilege

AI learns what each agent actually uses and auto-generates the least-privilege policy. No manual tuning.

✓

Fail-Closed Security

If policy engine can't verify, request is denied. Always.

Example: Agent Policy
# auto-generated by AI based on usage

enforcement:

audit: true
